# 📋 خطة تنفيذ التطوير المالي الشامل (FINANCIAL ERP 2026)

هذه الخطة هي المرجع التنفيذي لتحويل النظام المالي إلى نظام ERP محاسبي متكامل يتبع معايير دولة الإمارات (FTA) ومبدأ الاستحقاق.

---

## 🏗️ المرحلة 1: التأسيس والبنية التحتية (Foundation)
*الهدف: بناء "الدماغ" الذي يغذي النظام بالبيانات والربط المحاسبي.*

1. **تحديث قاعدة البيانات (Prisma Schema):**
   - [ ] إضافة موديل `FinancialSettings` (TRN, VAT%, Bank Accounts).
   - [ ] إضافة موديل `Invoice` و `InvoiceItem` لربط الرسوم بالضرائب.
   - [ ] إضافة موديل `PaymentInstallment` لجدولة الأقساط.
   - [ ] إضافة علاقات لربط المصاريف بأدلة الحسابات.

2. **محرك القيود الآلي (Automatic JE Engine):**
   - [ ] بناء وظيفة (Service) تقوم بتوليد قيود اليومية آلياً عند أي حركة مالية (فاتورة، سند، مصروف).

---

## ⚙️ المرحلة 2: الإعدادات المالية (Financial Settings)
*الهدف: ضبط الثوابت التي ستظهر في جميع الفواتير والتقارير.*

1. **واجهة الإعدادات:**
   - [ ] نموذج إدخال بيانات المنشأة القانونية و الـ TRN.
   - [ ] نموذج إدارة "خزائن النقد" وحسابات البنوك.
2. **الربط البرمجي:**
   - [ ] تخزين الـ VAT% كقيمة افتراضية (5%) ليتم سحبها تلقائياً عند الفوترة.

---

## 📄 المرحلة 3: الإيرادات والفواتير الضريبية (Tax Invoices)
*الهدف: تحويل "تسجيل الطالب" إلى استحقاق مالي قانوني.*

1. **محرك الفوترة:**
   - [ ] حساب الضريبة 5% على كل بند رسوم.
   - [ ] توليد رقم فاتورة تسلسلي قانوني.
2. **جدولة الأقساط:**
   - [ ] واجهة لتقسيم إجمالي الفاتورة على شهور السنة مع ربط حالة كل قسط.
3. **الأثر المحاسبي:**
   - [ ] ترحيل "قيد استحقاق" يثبت مديونية الطالب في الأستاذ العام.

---

## 💰 المرحلة 4: سندات القبض والدفع (Vouchers)
*الهدف: إثبات التدفق النقدي الفعلي وتحديث الأرصدة.*

1. **واجهة المقبوضات:**
   - [ ] اختيار الطالب -> عرض الأقساط المعلقة -> دفع مبلغ -> إغلاق القسط.
2. **واجهة المصاريف:**
   - [ ] تسجيل المصروف (إيجار، كهرباء) مع حساب ضريبة المدخلات ورفع صورة الفاتورة.
3. **الأثر المحاسبي:**
   - [ ] تحديث رصيد البنك/الخزينة فوراً وتخزين قيود اليومية.

---

## 👥 المرحلة 5: الرواتب وتكامل WPS (Payroll)
*الهدف: إدارة التزامات الموظفين وفق قانون العمل الإماراتي.*

1. **محاسبة الرواتب:**
   - [ ] ربط رواتب الموظفين (من قسم HR) بالدورة المحاسبية.
2. **نظام WPS:**
   - [ ] إصدار ملف SIF (Salary Information File) الخاص بالبنوك الإماراتية.

---

## 📊 المرحلة 6: التقارير والرقابة (Reporting)
*الهدف: استخراج النتائج النهائية للمديرين الماليين.*

1. **التقارير الضريبية:**
   - [ ] تقرير ضريبة القيمة المضافة (VAT Return) الجاهز للتقديم للهيئة.
2. **التقارير المحاسبية:**
   - [ ] ميزان المراجعة (Trial Balance).
   - [ ] قائمة الدخل (P&L) لبيان الربح والخسارة.
3. **الدليل المالي (Manual):**
   - [ ] صفحة توضيحية لترابط النظام (شرح التدفقات المالية).

---

## 🚀 خطة العمل الحالية (Next Steps)
1. **الآن:** تعديل `schema.prisma` لإضافة الإعدادات المالية.
2. **بعده:** بناء واجهة الإعدادات المالية.
3. **بعده:** ربط الفواتير والضريبة.
